# Boas Práticas da Equipe

Este documento reúne recomendações simples para manter o projeto organizado, seguro e fácil de manter.

## Recomendações

1. Nunca trabalhar diretamente na branch `main`.

2. Criar uma branch específica para cada alteração ou issue.

3. Fazer commits pequenos e com mensagens claras.

4. Atualizar a branch frequentemente com a versão mais recente da `main`.

5. Revisar as alterações antes de abrir um Pull Request.

6. Testar o projeto antes de enviar as mudanças.

7. Evitar subir arquivos sensíveis, como senhas, tokens ou chaves privadas.

8. Escrever descrições claras nos Pull Requests, informando o que foi alterado.

9. Manter a documentação atualizada quando houver mudanças importantes.

10. Resolver conflitos com atenção antes de finalizar um merge.
